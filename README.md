# 基于 RK3588 的多模态智能安全联锁系统

## 项目简介

本项目面向危化品库区临时动火检修等高风险作业场景，设计并实现了一套基于 RK3588 与 STM32 协同的工业视觉安全监控与联锁平台。系统采用“边缘智能识别 + 下位机安全联锁”双层架构，能够对现场人员、PPE、防护状态、火焰风险和环境异常进行综合判断，并联动完成分区断电、排风控制、语音报警、录像留存和云端同步。

## 系统特点

- 边缘侧完成实时视频采集与 AI 推理，减少云端依赖
- 双工作区独立识别，支持分区准入、分区断电和分区告警
- 火灾识别采用视觉与环境信息融合确认，降低误报
- STM32 负责执行机构控制，在通信异常时进入安全保护状态
- 支持 RTMP 推流、事件录像、断网缓存补发和 MQTT 状态同步
- Qt 上位机集中显示设备状态、报警信息和历史记录

## RK3588 端功能

RK3588 端是系统的边缘智能核心，主要承担以下任务：

1. 通过 V4L2 采集摄像头视频流
2. 运行 RKNN 模型完成目标检测
3. 识别两个红色工作区并进行人员区域归属判断
4. 识别 PPE 合规状态和多人干扰情况
5. 识别火焰目标，并结合环境状态确认火灾风险
6. 完成 RTMP 实时推流和事件录像分片
7. 将设备状态、报警状态和识别结果同步到 MQTT 云端
8. 与 STM32 串口通信，执行安全联锁控制

## STM32 端功能

STM32 端负责下位机安全执行，主要包括：

1. 采集温度、烟雾、可燃气体等环境信息
2. 采集复位按键和联锁输入
3. 控制工作区 1 / 工作区 2 电源
4. 控制风扇和报警输出
5. 在 RK3588 离线或通信异常时执行安全兜底

## Qt 上位机功能

Qt 上位机用于集中监控和结果展示，主要包括：

1. 实时环境监测界面
2. 工作区电源状态显示
3. 报警状态与设备状态展示
4. 火灾置信度曲线与最近报警记录
5. 录像回放与历史追溯

## 运行流程

系统启动后，RK3588 先获取视频并进行 AI 识别，再结合环境数据和区域信息进行融合判断。若检测到单人合规进入工作区，则允许对应电源开启；若出现多人干扰、区内 PPE 异常或火灾风险，则触发分区断电或全局联锁。所有状态会同步到 Qt 上位机和云端，方便集中查看和追溯。

## 部署说明

RK3588 端可直接使用工程目录下的编译脚本和部署脚本完成构建与烧录。Qt 上位机使用 Qt 工程构建后运行即可连接 MQTT 服务并显示设备状态。实际部署时，请根据现场环境配置串口、MQTT、推流地址和音频文件路径。
